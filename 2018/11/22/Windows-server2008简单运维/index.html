Windows server2008简单运维
发布于：2018-11-22 20:52:41 标签：/ 运维 / Windows server / 访问：

因为一些简单业务，有时候也需要我们程序猿亲自搭建一个项目运行环境，Windows server08就是一个不错的选择，资源可视化+环境集成，美滋滋~

接下来就开个Ws08开撸吧！

开始

首先你需要有一台运行着Windows server 2008的服务器，这里怎么开设我就不赘述了，直接开始上手。

开了机的我是一脸懵逼的，界面仿佛是千禧年的Windows 98，毫无美感的旧时代3d性冷淡桌面，系统及其纯净，基本上仅有我的电脑，回收站以及IE浏览器…

不过还好，我们有IE浏览器，可以下载网上的环境集成软件。现有的服务器系统集成软件有很多选择，比如护卫神、宝塔等等，这里我使用的是护卫神，其它环境集成软件基本类似…

环境从安装护卫神开始

打开IE浏览进入护卫神官网下载环境集成包，下载好后打开，一路下一步，具体环境根据自己的项目需求进行选择安装

之后就是护卫神的项目配置引导设置了，设置好项目根目录和域名后，护卫神就将网站为你配置完成了，并且提供给你一个txt文件告诉你项目所有信息（域名、FTP用户名、FTP密码、护卫神管理页面地址、护卫神管理员账号、密码、MySQL账号、密码等等）这时你已经可以使用护卫神为你配置好的网站了~

Windows server 2008 FTP中的坑

我发现服务器对于远程连接方面的策略真的是慎之又慎，甚至搞的我们非专业人士一头雾水，幸好护卫神已经帮我配置好了FTP，但是问题来了，护卫神为我们配置的FTP在本地连接localhost没问题，远程连接却是失败的。

接下来只要搞定FTP的这一坑，我们就能够使用Windows server创建一个完整的Web环境了。

我们知道，连接localhost就代表我们已经创建了一个可运行的FTP服务器，至于远程连接失败，极有可能是防火墙的安全策略屏蔽了我们的连接，善于Baidu + Google的我很快就找到解决办法：

  • 首先打开控制面板，找到windows防火墙
  • 打开windows防火墙后选择允许程序通过Windows防火墙
  • 选择添加程序
  • 点击浏览
  • 然后找到C:\WINDOWS\system32\svchost.exe这个程序完成添加即可~

OK！然后随便你创建什么项目啦，将文件传入根目录，没有域名可以直接通过访问IP进行浏览~
